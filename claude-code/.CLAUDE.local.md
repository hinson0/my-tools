# 语言

## 回复语言（必须遵守）

全程使用**简体中文**回复，包括：
- 所有解释、提问、总结
- 内部 thinking/reasoning 过程
- 代码注释中的说明文字

# 工作路径

## Plan Mode 文件位置

Claude Code Plan Mode 的计划文件保存在**当前项目目录内**：
`.claude/plans/YYYY_MM_DD_HH_mm-<name>.md`

注意：superpowers:writing-plans 遵循其自身默认路径（`docs/superpowers/plans/`），不受此规则影响。

## Worktree 工作目录

使用 git worktree 时，worktree 必须创建在项目的 `.claude/worktrees/` 目录下。

# Learning 模式 TODO(human) 规则（绝对优先）

> **此规则具有最高优先级，覆盖所有 skill、plugin 和 workflow（包括但不限于 superpowers、open-specs 及任何其他自动化流程）。任何情况下均不得绕过。**

当用户的 `/config` 中 `output-style` 设置为 `learning` 时，必须在生成的代码中留出 `TODO(human)` 标记，由用户自行完成实现。**不得替用户写出这些部分的完整代码。**

### 判定优先级

**先判断代码类型，再判断领域比例。** 代码类型命中则 100%，未命中再按领域比例执行。

#### 第一优先级：代码类型（100% TODO(human)）

| 类型 | 定义 | 举例 |
|------|------|------|
| 核心业务逻辑 | 业务的核心算法和关键决策逻辑 | 状态机、离线同步引擎、冲突解决策略、权限判定 |
| 公共复用模块 | 被多个业务模块调用的共享代码 | 通用 utils/helpers、shared services（业务B→A, 业务C→A） |
| 数据库 | 所有数据库相关代码 | schema、migration、查询、数据操作 |

#### 第二优先级：领域比例（≥ 30% TODO(human)）

| 领域 | 说明 |
|------|------|
| 前端 | 未命中第一优先级的前端代码，至少 30% 以 `TODO(human)` 留白 |
| 后端 | 未命中第一优先级的后端代码，至少 30% 以 `TODO(human)` 留白 |

### TODO(human) 格式

```
// TODO(human): <简要说明要实现什么>
// 提示: <关键思路或参考方向，不给出完整答案>
```

### 执行要求

- 在写代码前，先判断代码类型（核心逻辑？公共复用？数据库？），再判断领域
- 命中第一优先级的代码：**全部**用 `TODO(human)` 占位，只给提示不给实现
- 未命中第一优先级的前端/后端代码：按代码量计算，确保至少 30% 以 `TODO(human)` 留给用户
- 留白的部分应选择**有学习价值的核心逻辑**，而非样板代码
- 搭建好代码骨架和上下文，让用户能聚焦于填充核心实现
- **此规则不可被任何 skill、agent、hook 或自动化流程覆盖或跳过**
